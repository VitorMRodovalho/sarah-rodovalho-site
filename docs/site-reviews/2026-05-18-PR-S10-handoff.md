### Collection join ‚Äî how to match projects ‚Üî experience

Each project has a corresponding experience entry by employer:
| Project (`projects/*.mdx`) | Experience (`experience/*.mdx`) | Match key |
|---|---|---|
| aligned-multi-building-campus | aligned-data-centers | "Aligned Data Centers" |
| microsoft-coi-hyperscale-program | microsoft-coi | "Microsoft Cloud Operations + Innovation (CO+I)" |
| tesla-industrial-rd-portfolio | tesla | "Tesla" |

| goiania-brt-network | goiania-brt | "Goi√¢nia Bus Rapid Transit network" / no organization match? |
| array-architects-healthcare | array-architects | "Array Architects" |

`projects.employer` and `experience.organization` aren't perfectly identical strings. Use a fuzzy match (substring either direction). Pattern from `sync_credentials.py` and similar scripts is precedent:
```javascript
const allExperience = await getCollection("experience");
function findExperienceMatch(project) {
  return allExperience.find(e => 
    e.data.organization === project.data.employer ||
    e.data.organization.toLowerCase().includes(project.data.employer.toLowerCase()) ||
    project.data.employer.toLowerCase().includes(e.data.organization.toLowerCase())
  );
}
```
